Fixing a misidentified page type with Analyze API

Sometimes Diffbot’s Analyze API might misidentify a page as an unsupported type. In such cases, the page type might read other and not return any extracted content. There’s a way to override this.

Go to the Custom API UI and live preview the page being processed using the Analyze API:

Then, edit the type field by clicking on Edit:

Enter the literal value of the page type: product, article, video, image, and discussion.

Save and retry processing. Your rule will now see the page matching this rule’s domain regex as the type you defined.
